CSRF (Cross Site Request Forgery) 跨站请求伪造

# CSRF 简介

什么是 CSRF？
攻击者利用用户的身份操作用户账户的一种攻击方式

# CSRF 进阶

# 浏览器的 Cookie 策略

英文中文区别
Session Cookie临时 Cookie没有指定 Expire 时间，浏览器关闭后 Cookie 失效
Third-party Cookie本地 Cookie服务器在 Set-Cookie 指定了 Expire 时间，只有到了时间才会失效，该 Cookie 保存在本地

# P3P 头的副作用

P3P Header 是 W3C 指定的一项关于隐私的标准，全称 The Platform for Privacy Preferences


当请求www.b.com/test.html 时，它的 iframe 会告诉浏览器跨域请求 www.a.com/test.php

# GET?POST?

用户尝试构造 GET 请求
http://host/register?username=test&password=passwd
如果服务器没有对请求方法做限制，这个请求就会通过

# Flash CSRF


还可以使用 getURL，loadVars

# CSRF

Step1模拟服务器端取得 request 参数
Step2好友 json 数据的动态获取
Step3感染信息输出和消息发送的核心部分

# CSRF 的防御

# 验证码

因为 CSRF 攻击的过程中，通常是在用户不知情的情况下，添加验证码，强制用户与该网站进行交互，才能完成请求。但是验证码只能作为防御 CSRF 的辅助手段

# Referer Check

因为页面与页面之间有一定的逻辑，这使得 Referer 具有一定的规律
缺陷服务器并非什么时候都能取到 Referer

# Anti CSRF Token

# CSRF 的本质

重复操作的所有参数都是可以被攻击者猜测到的
解决方案参数加密或者使用随机数

# Token 的使用原则

需要使用安全的随机数生成 token
当前 token 使用过一次后，需要生成一个新的 token
使用 token 时需要注意 token 的保密性
尽量将 token 放在表单中，避免泄漏
